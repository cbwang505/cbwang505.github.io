ImageList Property (XtremeCalendarControl)

CalendarThemeOffice2007 Object  | BitmapBorders Property  | BitmapID Property

Xtreme Calendar ActiveX Control v15.3

ImageList Property

Description

Image collection used to set user bitmaps to customize some Calendar elements which are drawn using bitmap patterns.

Property type

Read-write property

Syntax (Visual Basic)

Public Property ImageList() As CalendarThemeImageList

Remarks

The ImageList collection is used to set user bitmaps to customize some Calendar elements which are drawn using bitmap patterns.  These images are headers (day, group, week,...) and scroll/expand day buttons.

 

Note: Only headers customization is supported for now, but other objects customization may be added later in the same way. 

 

Header background is drawn in 2 steps: 

1. Fill header rectangle using base color. 

2. Draw pattern bitmap in header rectangle (Bitmap may be an alpha bitmap as well). 


For the majority of Calendar headers default bitmaps are 'grey' alpha bitmaps. When they are drawn over a 'base color' we got these nice headers. We can simply change a 'base color' (and don't touch bitmap) to change header color style. 


CalendarThemeHeaderBackground contains 2 properties: 

       BitmapID - contains ID of a bitmap from CalendarThemeOffice2007.ImageList. 

               Standard (default) bitmaps have negative IDs like -100, -101. 

               Custom bitmaps should have positive IDs. 

               Zero ID (0) is used to set 'no bitmap'. 


       BitmapBorders - Template bitmap has some constant size and it is resized to header rect when drawn. This member contains bitmap margins which are not resized. For example if a bitmap for a header has 1 pixel frame around, we have to set BitmapBorders as (1, 1, 1, 1). This will prevent resizing of border frame width. For any header size - border will be 1 pixel width. 

 

Header bitmap is divided by 6 equal height pieces. 

First triad is for a normal header, second triad is for a hot header (when mouse is over a header);


First piece (in each triad) is used for a left header (in header row); 

second piece (in each triad) is used for a middle header (in header row); 

third piece (in each triad) is used for a right header (in header row); 


When only one header in a row: 

       the left header part is drawn using the left half of the first piece; 

       the right header part is drawn using the right half of the third piece;
