MarkupEnabled Property

CalendarControl Control

Xtreme Calendar ActiveX Control v15.3

MarkupEnabled Property

Description

Allows the use of a XAML snippet to be used to display text in events\appointments.

Property type

Read-write property

Syntax (Visual Basic)

Public Property MarkupEnabled() As Boolean

Remarks

 

The Calendar allows you to use XAML to format the contents of events\appointments displayed in the calendar. Only the Office 2007 calendar theme supports XAML.

There is a "static" Markup use case and "dynamic" one. Both of them works only for Office 2007 theme.

For Static you only to have enable Markup in the Calendar and it will show markup text automatically for those events which have "xtpMarkupTemplate" property set (see in code).

For Dynamic you have to enable text customization (GetItemText method), and then set Markup text for some events...

The following steps are needed to enable XAML support:

  • Set the theme to Office 2007 (Both static and dynamic uses)
  • Set MarkupEnabled to True (Both static and dynamic uses)
  • Set the xtpCalendarItemText_EventMarkupTemplate flag (Dynamic use only)


First the Office 2007 theme must be used. Second, you must tell the Calendar to enable XAML support. Third, if you plan to change XMAL text dynamically you need the AskItemTextFlags method to set the xtpCalendarItemText_EventMarkupTemplate flag.

(Static use) If all you want to do is display XAML text for each event statically, you simply add a custom property nammed "xtpMarkupTemplate" and the calendar will display the XAML.

(Dynamic use) AskItemTextFlags specify that the GetItemText event will fire when it comes time to draw the text for an event\appointment. We only specified the xtpCalendarItemText_EventMarkupTemplate flag so we will only be notified when we need to set the XAML text for the event\appointment.

(Dynamic use) When using the xtpCalendarItemText_EventMarkupTemplate, you will be creating all viewable text for the event\appointment using XAML. You don't need a subject, location, body, etc. It will only display the XAML string you provide, which can be any valid XAML snippet.

(Static and Dynamic use) The best way to store a XAML snippet is in the custom properties of an event\appointment. With this method you can add as many snippets as you need.

When using statically, you simply add a property nammad "xtpMarkupTemplate" and the string of the property should be the XAML string to display in the event.

Note that you might not want to display the XAML string when in WeekView or MonthView depending on how much XAML was added or the size of the displayed XAML. In this case you could simply test for these views in the GetItemText and choose not to use the XAML for those views.

The sample below shows how to enable Markup support in the Calendar and how to add XAML to an event\appointment.

This sample can be used both statically and dynamically, just un-comment the code specified for dynamic use.

Note that this can be used as a stand alone application, simply cut and paste all of the code and place it inside the code section of a form that contains a calendar control:

Example

[Visual Basic] 

Private Sub Form_Load()
    'Use Office 2007 Theme for Calendar
    CalendarControl.VisualTheme = xtpCalendarThemeResource

    'Enable Markup
    CalendarControl.MarkupEnabled = True

    'Enable custom text handling for event
    'Un-comment this line for dynamic use
    'CalendarControl.AskItemTextFlags = xtpCalendarItemText_EventMarkupTemplate

    'Set the Calendar's View
    Dim Today As Date
    Today = Now
    CalendarControl.ViewType = xtpCalendarDayView
    CalendarControl.DayView.ShowDay DateTime.Date

    CalendarControl.DataProvider.AddEvent Add_Test_Event

    CalendarControl.Populate
    CalendarControl.DayView.ScrollToWorkDayBegin
End Sub

'Ony used when text is added dynamically
Private Sub CalendarControl_GetItemText(ByVal Params As _
              XtremeCalendarControl.CalendarGetItemTextParams)

    If Params.Item = xtpCalendarItemText_EventMarkupTemplate Then
        If CalendarControl.MarkupEnabled And _
                           Params.ViewEvent.Event.CustomProperties.Count Then
            'Here you can add some custom XAML, it does not need to be stored
            'in the properties of an event
            Params.Text = _
            Params.ViewEvent.Event.CustomProperties.Property("xtpMarkupTemplate")
        End If
    End If

End Sub

'Function to create a sample event
Private Function Add_Test_Event() As CalendarEvent
        Dim m_pEditingEvent As CalendarEvent

        Set m_pEditingEvent = CalendarControl.DataProvider.CreateEvent

        m_pEditingEvent.StartTime = Now
        m_pEditingEvent.EndTime = Now + 0.22

        'Properties of the event, they will be displayed when
        'Markup is not enabled.
        m_pEditingEvent.Subject = "SuitePro 2009 Released!"
        m_pEditingEvent.Location = "Internet Download (Link Included)"
        m_pEditingEvent.Body = "Codejock Software released SuitePro" & _
                               " 2009 today. All development machines " & _
                               "must be updated no later than 06/15/08." & _
                               "Implementation will begin immediately!"
        m_pEditingEvent.AllDayEvent = False
        m_pEditingEvent.Label = 2
        m_pEditingEvent.BusyStatus = xtpCalendarBusyStatusFree
        m_pEditingEvent.MeetingFlag = True
        m_pEditingEvent.ScheduleID = 1

        'Store a XAML string to be displayed in the event body
        m_pEditingEvent.CustomProperties.Property("xtpMarkupTemplate") = _
"<StackPanel Margin='5, 5, 5, 5' VerticalAlignment='Top'" & _
     "HorizontalAlignment='Center'>" & _
"<TextBlock Padding='0,10,0,0' TextWrapping='Wrap'><Bold>Subject:</Bold> " & _
            "<Bold>SuitePro</Bold> <Run Foreground='White'>2009</Run> " & _
            "Released!</TextBlock>" & _
"<TextBlock Padding='0,10,0,0'><Bold>Body:</Bold></TextBlock>" & _
"<TextBlock Padding='10,10,0,0' TextWrapping='Wrap'>" & _
    "<Bold><Run Foreground='Blue'>Codejock</Run> Software</Bold> released " & _
    "<Bold>SuitePro</Bold> <Run Foreground='White'>2009</Run> today. All" & _
    " development machines <Bold>must</Bold> be updated no later than " & _
    "<Run Foreground='Red'>06/15/08</Run>." & _
"</TextBlock>" & _
"<TextBlock Padding='10,10,0,0' TextWrapping='Wrap'>" & _
    "Download the software <Hyperlink Cursor='Hand'>here</Hyperlink>." & _
"</TextBlock>" & _
"<TextBlock Padding='10,10,0,0' TextWrapping='Wrap'>" & _
    "Implementation will begin immediately! If you have and comments or" & _
    "concerns email John at <Hyperlink Foreground='Blue' Cursor='Hand'>" & _

"<TextBlock Padding='10,10,0,0'>" & _

"</TextBlock>" & _
"</StackPanel>"

        Set Add_Test_Event = m_pEditingEvent
End Function
